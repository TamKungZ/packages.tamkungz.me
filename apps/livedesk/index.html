livedesk

Live wallpaper for GNOME

Livedesk turns a video file into a GNOME desktop background. A GNOME Shell extension places the wallpaper actor inside the compositor, a small Rust + GStreamer daemon decodes the video, and a GTK4/Libadwaita app handles configuration. Unlike a hidden fullscreen player pretending to be a wallpaper, it draws inside GNOME Shell's own background group — behind normal windows, not underneath them.

##features

  • Renders as a real GNOME Shell background actor — normal windows sit on top of it like any wallpaper.
  • livedesk is a GTK4/Libadwaita app for picking videos, monitors, and playback state.
  • Frames move through a memory-mapped file guarded by a seqlock header — D-Bus only carries coarse commands.
  • D-Bus controls for play, pause, mute, source switching, and frame-path lookup.
  • Two extension builds — ES module and legacy imports.* — cover GNOME Shell 40 through 51.
  • Scripted Debian and RPM packages, plus a Launchpad PPA pipeline.

##architecture

video file
  -> livedesk-daemon
  -> /run/user/<uid>/livedesk/<monitor>.frame
  -> GNOME Shell extension
  -> Clutter actor in Main.layoutManager._backgroundGroup
  -> Mutter compositor

No pixel data crosses D-Bus. The daemon writes RGBA frames to a memory-mapped file; the Shell extension polls it using a small seqlock header to avoid torn reads. D-Bus carries only coarse control: play, pause, mute, source changes, frame-path lookup.

worth knowing — background-actor placement and fullscreen/lock-screen detection rely on GNOME Shell internals rather than a stable extension API, so a Shell update can occasionally need a fix. The current uploader also goes through CPU memory rather than a zero-copy GPU path — fine today, a candidate for a future DMABUF/GL route at very high resolutions or frame rates.

##install

Package installs place the extension automatically for the detected Shell version. Building from source works everywhere else.

# via PPA
sudo add-apt-repository ppa:tamkungz/stable
sudo apt update
sudo apt install livedesk
livedesk

# or add the APT repo manually
curl -fsSL https://packages.tamkungz.me/gpg.key \
  | sudo gpg --dearmor -o /usr/share/keyrings/tamkungz-archive-keyring.gpg
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/tamkungz-archive-keyring.gpg] https://packages.tamkungz.me/apt stable main" \
  | sudo tee /etc/apt/sources.list.d/tamkungz.list
sudo apt update
sudo apt install livedesk
livedesk
sudo rpm --import https://packages.tamkungz.me/gpg.key
sudo tee /etc/yum.repos.d/tamkungz.repo >/dev/null <<'EOF'
[tamkungz]
name=TamKungZ Packages
baseurl=https://packages.tamkungz.me/rpm/x86_64
enabled=1
gpgcheck=1
repo_gpgcheck=1
gpgkey=https://packages.tamkungz.me/gpg.key
EOF
sudo dnf install livedesk
livedesk
./install.sh
livedesk

On Wayland, log out and back in after a fresh extension install. On X11, restart GNOME Shell with Alt+F2, then r, then Enter. The installer picks the ES-module extension for GNOME 45–51 or the legacy imports.* build for GNOME 40–44 automatically.

##requirements

To build from source. The *-dev packages compile the Rust daemon; the gstreamer1.0-* plugins provide runtime decoders.

sudo apt install cargo rustc pkg-config gjs gir1.2-gtk-4.0 gir1.2-adw-1 \
  libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev libdbus-1-dev \
  libunwind-dev \
  gstreamer1.0-plugins-good gstreamer1.0-plugins-bad gstreamer1.0-libav \
  totem
  • totem supplies totem-video-thumbnailer, used for gallery thumbnails when present.
  • Plugin packages cover H.264, VP8, VP9, and AV1 decoding at runtime.
  • Missing gstreamer-1.0.pc or similar .pc files usually mean the runtime is installed but headers aren't.

fix for missing headers — sudo apt install libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev libunwind-dev

##usage

Open anytime with livedesk. The hamburger menu in the top-right holds daemon controls and preferences.

  • Library gallery — browse the video library as thumbnails and apply a pick immediately.
  • Import & organize — open the library folder directly, or import files into it.
  • Per-monitor targeting — detects connected monitors and matches their resolution automatically.
  • Playback controls — mute, unmute, start the daemon, send play / pause / stop / apply over D-Bus.
video library~/Videos/Livedesk
thumbnail cache~/.cache/livedesk/thumbnails
config file~/.config/livedesk/config.json

What it does

Livedesk writes a video URI to GNOME's own org.gnome.desktop.background picture-uri and picture-uri-dark settings. The native GNOME Shell overlay detects video wallpaper URIs and asks livedesk-daemon to decode frames into $XDG_RUNTIME_DIR/livedesk.

It is not a hidden fullscreen video player. Normal windows sit above the live background the same way they sit above a normal wallpaper.

Support status

The current native setup targets GNOME Shell 40-44 and has been tested on GNOME Shell 43.x. GNOME Shell 45+ changed its JavaScript module layout and needs a separate native overlay before it is treated as supported.

APT packages are available for amd64 and arm64. RPM packages are published under architecture-specific repos, so the recommended RPM base URL uses $basearch.

After install

Run livedesk-setup once, log out and back in, then open livedesk. Put videos in ~/Videos/Livedesk, double-click a thumbnail, or select a video and click Save and Apply.

The app can import videos, generate thumbnails, control service mode/autostart, start an Ayatana/AppIndicator tray helper, restore the normal wallpaper, and write detailed logs to ~/.cache/livedesk/livedesk.log.

Emergency recovery

Native setup changes how GNOME Shell loads JavaScript on the next login. If GNOME shows Oh no! Something has gone wrong., open a TTY with Ctrl+Alt+F3, log in, and run:

shTTY recovery
rm -f ~/.config/environment.d/90-livedesk-gnome-shell.conf
rm -rf ~/.local/share/livedesk/gnome-shell-js
sudo systemctl restart gdm

If the distro does not use GDM, run sudo reboot. Return to the graphical session with Ctrl+Alt+F1 or, on some systems, Ctrl+Alt+F2.
